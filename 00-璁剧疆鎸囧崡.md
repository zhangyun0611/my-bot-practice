# 三机器人闭环 — 完整设置指南

## 整体架构

```
┌─────────────┐     提 Issue      ┌─────────────┐
│  审核机器人   │ ──────────────→  │   GitHub     │
│  (audit.ps1) │                  │   Issues     │
└─────────────┘                  └──────┬──────┘
      ↑                                 │
      │                                 ↓
┌─────────────┐     提 PR        ┌─────────────┐
│  审查机器人   │ ←────────────── │  修复机器人   │
│ (review.ps1) │                  │  (fix.ps1)   │
└──────┬──────┘                  └─────────────┘
       │
       ↓
    合并 PR，关闭 Issue
```

每个机器人就是一个 PowerShell 脚本，用 Windows 任务计划程序定时跑。

---

## 第一步：安装 GitHub CLI

打开 PowerShell（管理员），运行：

```powershell
winget install GitHub.cli
```

装完后关掉 PowerShell 重新打开，然后登录：

```powershell
gh auth login
```

选择：
- GitHub.com
- HTTPS
- Login with a web browser

浏览器会弹出来，授权就行。

验证一下：

```powershell
gh auth status
```

看到 "Logged in to github.com" 就对了。

---

## 第二步：创建练手项目

在 GitHub 上创建一个新仓库，然后克隆到本地：

```powershell
# 在 GitHub 上创建仓库（公开的，方便操作）
gh repo create my-bot-practice --public --clone

# 进入项目目录
cd my-bot-practice
```

把 `practice-project` 文件夹里的文件全部复制进去，然后推上去：

```powershell
git add -A
git commit -m "init: 练手项目初始化"
git push
```

---

## 第三步：把三个脚本放到项目里

把这三个文件复制到项目根目录的 `_bots/` 文件夹：

```
my-bot-practice/
├── _bots/
│   ├── audit.ps1      ← 审核机器人
│   ├── fix.ps1        ← 修复机器人
│   └── review.ps1     ← 审查机器人
├── src/
│   └── ...            ← 你的代码
└── ...
```

---

## 第四步：手动测试一轮

先别急着设定时任务，手动跑一遍确认没问题：

```powershell
cd C:\你的路径\my-bot-practice

# 第1步：审核（会创建 GitHub Issue）
powershell -File _bots\audit.ps1

# 等它跑完，去 GitHub 看看有没有新 Issue
gh issue list

# 第2步：修复（会针对 Issue 创建 PR）
powershell -File _bots\fix.ps1

# 等它跑完，去 GitHub 看看有没有新 PR
gh pr list

# 第3步：审查合并
powershell -File _bots\review.ps1
```

## 第五步：设置定时任务（自动循环）

用 Windows 任务计划程序，让三个脚本错开时间跑：

| 机器人 | 频率 | 时间 |
|--------|------|------|
| 审核 audit.ps1 | 每天2次 | 08:00, 20:00 |
| 修复 fix.ps1 | 每天2次 | 09:00, 21:00 |
| 审查 review.ps1 | 每天2次 | 10:00, 22:00 |

错开1小时，让前一个跑完后一个再跑。

### 用命令行创建定时任务：

```powershell
# 审核机器人 - 每天 8:00 和 20:00
schtasks /create /tn "BotLoop-Audit" /tr "powershell -ExecutionPolicy Bypass -File C:\你的路径\my-bot-practice\_bots\audit.ps1" /sc daily /st 08:00
schtasks /create /tn "BotLoop-Audit-PM" /tr "powershell -ExecutionPolicy Bypass -File C:\你的路径\my-bot-practice\_bots\audit.ps1" /sc daily /st 20:00

# 修复机器人 - 每天 9:00 和 21:00
schtasks /create /tn "BotLoop-Fix" /tr "powershell -ExecutionPolicy Bypass -File C:\你的路径\my-bot-practice\_bots\fix.ps1" /sc daily /st 09:00
schtasks /create /tn "BotLoop-Fix-PM" /tr "powershell -ExecutionPolicy Bypass -File C:\你的路径\my-bot-practice\_bots\fix.ps1" /sc daily /st 21:00

# 审查机器人 - 每天 10:00 和 22:00
schtasks /create /tn "BotLoop-Review" /tr "powershell -ExecutionPolicy Bypass -File C:\你的路径\my-bot-practice\_bots\review.ps1" /sc daily /st 10:00
schtasks /create /tn "BotLoop-Review-PM" /tr "powershell -ExecutionPolicy Bypass -File C:\你的路径\my-bot-practice\_bots\review.ps1" /sc daily /st 22:00
```

> 记得把 `C:\你的路径\` 换成你实际的路径！

## 费用估算

每次审核 + 修复 + 审查，大概消耗 10k-50k tokens，取决于项目大小。

- 一天跑2轮 ≈ $1-5/天
- 用 Sonnet 比 Opus 便宜很多，练手阶段推荐 Sonnet

## 常见问题

**Q: Claude Code CLI 怎么非交互式运行？**
用 `--print` 参数（只输出结果，不进入交互模式）：
```powershell
claude --print "你的指令"
```

**Q: 怎么停掉自动循环？**
```powershell
schtasks /delete /tn "BotLoop-Audit" /f
schtasks /delete /tn "BotLoop-Audit-PM" /f
# ... 把6个都删掉
```

**Q: 怎么看运行日志？**
三个脚本都会把日志写到 `_bots/logs/` 目录下。
